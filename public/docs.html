Users · Admin Dashboard

Per-user analytics view aggregating data from user profiles, crop scans, and orders. One row per user. Answer: "Who is using our platform, what are they doing, and are they converting?"

const sm = JSON.parse(localStorage.getItem("mercbex_admin_scan_map") || "[]");
const ids = new Set(sm.filter((m) => m.uid === u.id).map((m) => m.sid));
const userScans = scans.filter((s) => ids.has(s.id));

Implementation

Grouping

const groups = new Map();
ordersList.forEach((o) => {
  if (!groups.has(o.customer.phone)) groups.set(o.customer.phone, []);
  groups.get(o.customer.phone)!.push(o);
});

Crop Scans · Admin Dashboard

Tracking the AI scan lifecycle — new scan → incomplete → action needed → in cart — with user attribution.

Scan Lifecycle

1
Incomplete
User confirmed ≥1 disease but left before viewing treatment.
2
Action Needed
User saw treatment but didn't add to cart.
3
In Cart
User added ≥1 product. Highest engagement.

Status Mapping

EventStatusCategory
Results shownnew_scanInitial
"Save & See Treatment"action_neededAction Needed
Add to Cartin_cartIn Cart
Rescan without savingincompleteIncomplete
All rejected / no confirmNot saved—
