01 — User Stories & Acceptance Criteria

Format: As a [role], I want [action], so that [benefit].
Each story includes acceptance criteria, priority, epic, and related API endpoints.

Epic: Authentication & Onboarding

US-001 P0 — Critical Auth
Phone Number Login via OTP
As a farmer,
I want to log in using my phone number and an OTP,
so that I don't need to remember a password and can access the app easily.
Acceptance Criteria
  • Farmer enters a valid 10-digit Indian phone number with country code
  • System sends a 6-digit OTP to the phone number within 5 seconds
  • OTP expires after 5 minutes
  • Farmer can request a new OTP after 30 seconds (retry cooldown)
  • After 3 failed OTP attempts, the session is locked for 15 minutes
  • Successful verification returns access token + refresh token
  • If phone number is new, a user account is automatically created
  • Token persists across browser sessions (localStorage)
Related APIs
POST /auth/send-otp POST /auth/verify-otp POST /auth/refresh
US-002 P2 — Medium Auth
Session Persistence & Auto-Logout
As a farmer,
I want to stay logged in when I return to the app,
so that I don't have to log in every time I open it.
Acceptance Criteria
  • Access token is valid for 24 hours
  • Refresh token is valid for 30 days
  • App silently refreshes the access token using the refresh token when expired
  • If refresh token expires, redirect to login screen
  • Logout invalidates both tokens server-side
Related APIs
POST /auth/refresh POST /auth/logout

Epic: Crop Selection

US-003 P0 — Critical Crops
Select My Crop
As a farmer,
I want to select which crop I'm growing from a visual grid,
so that the AI scanner knows what to look for when I scan.
Acceptance Criteria
  • Crop grid displays all supported crops with thumbnail images
  • Crops are organized in tiers: Tier 1 (featured), Tier 2 (secondary), Tier 3 (other)
  • Each crop card shows name, image, and top 2 disease tags
  • Only ONE crop can be selected at a time
  • Selected crop persists in local storage across sessions
  • Changing crop from home page and navigating to scanner starts a fresh scan
  • Previous incomplete scan is archived as "Incomplete" in My Diagnoses
  • Search bar allows filtering crops by name
Related APIs
GET /crops GET /crops/{crop_id}

Epic: AI Crop Scanning

US-004 P0 — Critical Scanning
Capture or Upload Crop Photo
As a farmer,
I want to take a photo of my crop using the camera or upload from gallery,
so that the AI can analyze it for diseases.
Acceptance Criteria
  • Camera opens with live viewfinder on mobile devices
  • Capture button takes a photo and shows preview
  • Gallery upload accepts JPEG and PNG (max 10MB)
  • Image is compressed to thumbnail (320px, 50% quality) for session storage
  • Full-resolution image is uploaded to backend for analysis
  • Camera permission is requested on first use with clear explanation
  • If camera is denied, only gallery upload option is available
Related APIs
POST /scan/analyze
US-005 P0 — Critical Scanning
AI Disease Analysis
As a farmer,
I want to see a progress indicator while my crop photo is being analyzed,
so that I know the system is working and roughly how long it will take.
Acceptance Criteria
  • Progress bar shows 0-100% during analysis
  • Rotating status messages provide context ("Scanning leaf patterns...", "Checking for fungal infections...")
  • Scanned photo preview is visible during analysis
  • Analysis completes within 3-5 seconds (Plantix API SLA)
  • If Plantix returns poor image quality feedback (focus: "bad" or distance: "too_far"), show retake guidance
  • If Plantix API times out (>10s), show error with retry option
  • Results are saved to scan session immediately on completion
Related APIs
POST /scan/analyze Plantix POST /v2/image_analysis
US-006 P0 — Critical Scanning
View Diagnosis Results with Reference Images
As a farmer,
I want to see what diseases were detected in my crop, with reference images I can compare against my photo,
so that I can validate whether the AI diagnosis is accurate.
Acceptance Criteria
  • Header shows total diseases detected (e.g., "3 issues detected in Rice")
  • Disease cards are sorted by severity: Critical → High → Medium → Low
  • Most severe disease card is auto-expanded; others are collapsed
  • Each expanded card shows: disease name, severity pill, likelihood badge
  • 3 reference images from Plantix are displayed alongside the farmer's scanned photo
  • Symptoms are shown as a bullet list (from symptoms_short)
  • Cause/trigger is displayed
  • Preventive measures are listed
  • "Healthy" diagnosis (peat_id 999999) is filtered out — not shown as a disease
  • If crop_health is "healthy", show a success state instead of disease cards
Related APIs
POST /scan/analyze (response)
US-007 P0 — Critical Scanning
Confirm Diagnosis & See Treatment
As a farmer,
I want to confirm that a detected disease matches my crop before seeing medicines,
so that I only buy treatments for problems I actually have.
Acceptance Criteria
  • Each disease card has a "Confirm & See Treatment" button
  • Button is only visible BEFORE confirmation
  • Medicines/products are hidden until the farmer confirms
  • After clicking confirm, the card expands to show recommended products
  • Confirmation is saved to the scan session and backend
  • Confirmed diseases show a green checkmark on the card header
  • Each disease can be confirmed independently
  • A "Reject" option is also available if the farmer disagrees
Related APIs
POST /scan/{scan_id}/confirm POST /scan/{scan_id}/reject
US-007B P1 — High Scanning
Reject a Diagnosis That Doesn't Match
As a farmer,
I want to reject a detected disease if it doesn't match my crop after comparing with reference images,
so that I'm not shown irrelevant treatment recommendations and the AI can learn from my feedback.
Acceptance Criteria
  • A "Reject" button is shown next to "Confirm & See Treatment" on each disease card
  • Clicking "Reject" shows a reason picker with options: "Doesn't match reference images", "Wrong crop detected", "Symptoms look different", "Other reason"
  • A "Cancel" link returns to the confirm/reject buttons without rejecting
  • After selecting a reason, the disease card is dimmed with reduced opacity and a strikethrough on the disease name
  • Rejected card shows a "Rejected" badge instead of severity pill
  • Rejected card cannot be expanded again
  • No medicines or prevention tips are shown for rejected diseases
  • Rejected diseases are NOT saved to My Diagnoses history (excluded from archive)
  • After rejecting, the next unresolved disease card auto-expands
  • Rejection is saved to the scan session in localStorage
  • Rejection reason is sent to the backend for AI feedback/training
Related APIs
POST /scan/{scan_id}/reject
US-008 P0 — Critical Scanning
Add Recommended Medicines to Cart
As a farmer,
I want to add recommended medicines to my cart with a specific quantity,
so that I can purchase the right treatment for my crop disease.
Acceptance Criteria
  • Each recommended product shows: image, name, brand, price, dosage instructions
  • Quantity selector allows incrementing/decrementing (min: 1)
  • "Add to Cart" button adds the product with the selected quantity
  • After adding, button changes to "✓ Added" (green) to prevent duplicate adds
  • Cart item includes scan context: crop name, crop icon, disease name, severity, scan timestamp
  • Adding a product updates the scan status to "In Cart" in My Diagnoses
  • Both chemical and organic treatment options are shown when available
Related APIs
POST /cart/items

Epic: Diagnosis History & Session Management

US-009 P1 — High Diagnoses
View My Diagnosis History
As a farmer,
I want to see a list of all my past crop scans on the home page with their current status,
so that I know which diagnoses still need action.
Acceptance Criteria
  • "My Diagnoses" section appears on the home page only if scan history exists
  • Each card shows: crop icon, crop name, disease name, severity pill, time ago
  • Status pills: Incomplete (left before confirming), Action Needed (confirmed but no medicines added), In Cart (medicines added to cart)
  • Cards are sorted by most recent first
  • Maximum 50 scan records are stored
  • Tapping a card navigates to /scan and resumes the session
Related APIs
GET /scans GET /scans/{scan_id}
US-010 P1 — High Diagnoses
Resume an Incomplete Scan
As a farmer,
I want to continue a scan from where I left off,
so that I don't have to retake the photo and wait for analysis again.
Acceptance Criteria
  • Scan session is saved to localStorage on every state change
  • Session stores: selected crop, results, compressed image thumbnail, confirmed diseases, added products, quantities
  • When navigating to /scan, if a saved session exists for the SAME crop, it restores the last state
  • Photo, results, confirmations, and added products are all restored
  • If the saved session is for a DIFFERENT crop, it archives the old session and starts fresh
  • Archived sessions appear as "Incomplete" in My Diagnoses
  • Image is compressed to 320px/50% quality to fit localStorage limits
Related APIs
GET /scans/{scan_id}
US-011 P2 — Medium Diagnoses
Scan Another Crop
As a farmer,
I want to start a new scan for a different crop after finishing one,
so that I can diagnose multiple crops in one session.
Acceptance Criteria
  • "Scan Another Crop" button is visible on the results page
  • Clicking it archives the current session (Incomplete/Action Needed/In Cart)
  • All scan state is cleared (photo, results, confirmations)
  • User returns to crop selection step
  • Previous scan appears in My Diagnoses on home page

Epic: Cart & Checkout

US-012 P0 — Critical Cart
Cart Grouped by Crop with Scan Context
As a farmer,
I want to see my cart items grouped by crop, with clear info about which disease each medicine treats,
so that I understand why each product was recommended and feel confident in my purchase.
Acceptance Criteria
  • Cart items from AI scan are grouped under crop headers (e.g., "🌾 Rice")
  • Each item shows: product name, price, quantity, crop icon + disease name + scan timestamp
  • Items added directly from Products page (no scan) appear under "Other Items"
  • Quantity can be updated with +/- buttons
  • Items can be removed individually
  • Cart total, subtotal, and discount are calculated correctly
  • Empty cart shows a CTA to scan a crop or browse products
Related APIs
GET /cart PATCH /cart/items/{id} DELETE /cart/items/{id}
US-013 P0 — Critical Cart
Place an Order
As a farmer,
I want to place an order with my delivery address and payment method,
so that the medicines are delivered to my village.
Acceptance Criteria
  • Checkout form collects: name, phone, address (line1, line2, district, state, pincode, landmark)
  • Cash on Delivery (COD) is the default payment method
  • Order confirmation shows: order ID, items summary, estimated delivery date
  • Cart is cleared after successful order placement
  • Scan status updates to "Completed" for associated diagnoses
  • Order confirmation is sent via SMS to the farmer's phone
Related APIs
POST /orders POST /user/addresses
US-014 P1 — High Cart
Track My Order
As a farmer,
I want to see the delivery status of my order,
so that I know when my medicines will arrive.
Acceptance Criteria
  • Order detail page shows: status (Confirmed → Packed → Shipped → Delivered), tracking number, carrier
  • Timeline shows each status event with timestamp
  • Estimated delivery date is displayed prominently
  • Order can be cancelled before shipping
  • Cancellation reason is collected (optional notes)
  • Order history shows which crops/diseases the medicines were for
Related APIs
GET /orders GET /orders/{order_id} POST /orders/{order_id}/cancel

Epic: Product Catalog

US-015 P1 — High Products
Browse & Search Products
As a farmer,
I want to browse and search the product catalog by category, crop, or disease,
so that I can find the right pesticide even without scanning.
Acceptance Criteria
  • Products page shows all products in a grid/list view
  • Filter by category: Insecticide, Fungicide, Herbicide, Growth Promoter, Organic
  • Filter by crop: shows only products relevant to selected crop
  • Filter by disease: shows only products that treat a specific disease
  • Search by product name or brand
  • Sort by: price, rating, popularity, newest
  • Each product card shows: image, name, brand, price (with discount if any), rating
  • Pagination or infinite scroll for large catalogs
Related APIs
GET /products GET /products/{product_id}

US-016 P2 — Medium Profile
Manage My Profile & Addresses
As a farmer,
I want to update my name, language preference, and save delivery addresses,

  • Profile shows: name, phone (read-only), email (optional), language, selected crops

  • Language preference affects app content (Hindi, English, Telugu, etc.)
  • Multiple delivery addresses can be saved
  • One address can be marked as default
  • Address fields: name, phone, line1, line2, district, state, pincode, landmark

02 — General Information

Base URL

https://api.mercbex.com/v1

Authentication

All authenticated endpoints require a Bearer token in the Authorization header.
Tokens are obtained via the /auth/login or /auth/verify-otp endpoints.
Standard Headers
{
  "Authorization": "Bearer <access_token>",
  "Content-Type": "application/json",
  "Accept": "application/json",
  "X-App-Version": "1.0.0",
  "X-Platform": "web | android | ios"
}

Standard Error Response

{
  "success": false,
  "error": {
    "code": "INVALID_INPUT",

05 — AI Scan & Detection APIs

Architecture: The scan flow uses Plantix Vision API as the external AI engine. Our backend acts as a proxy — it receives the farmer's image, forwards it to Plantix, then enriches the Plantix response with our product catalog (treatment recommendations) before returning to the frontend.

  Frontend                    MERCBEX Backend                  Plantix API
  ────────                    ──────────────                  ───────────
      │                            │                              │
      │  POST /scan/analyze        │                              │
      │  (image + crop_id)         │                              │
      │ ─────────────────────────► │                              │
      │                            │  POST /v2/image_analysis     │
      │                            │  (image + crop + coords)     │
      │                            │ ────────────────────────────►│
      │                            │                              │
      │                            │  Plantix Response            │
      │                            │  (diagnoses + ref images)    │
      │                            │ ◄────────────────────────────│
      │                            │                              │
      │                            │  ENRICHMENT:                 │
      │                            │  1. Filter out "Healthy"     │
      │                            │     if peat_id == 999999     │
      │                            │  2. Map likelihood →         │
      │                            │     severity                 │
      │                            │  3. Map pathogen_class →     │
      │                            │     MERCBEX product catalog  │
      │                            │  4. Store scan record in DB  │
      │                            │  5. Store image in CDN       │
      │                            │                              │
      │  Enriched Response         │                              │
      │  (diagnoses + products)    │                              │
      │ ◄───────────────────────── │                              │
      │                            │                              │
Mapping Rules
Plantix FieldMERCBEX FieldMapping Logic
diagnosis_likelihoodseverityvery_likely → critical/high, likely → high/medium, possible → medium/low
pathogen_classProduct category filterfungus → fungicides, pest → insecticides, abiotic → growth promoters
peat_idDisease mappingMap Plantix disease IDs to MERCBEX product catalog
image_referencesreference_imagesPass through, replace __w-100-200-400-600-800-1000__ with __w-600__ for optimal size
peat_id == 999999—Filter out — this means "Healthy", don't show as disease
image_feedback.focus == "bad"—Return error asking farmer to retake photo
language (from request)Accept-Language headerPass farmer's language preference to Plantix. Disease names, symptoms, treatment text returned in that language. Fall back to user profile language, then "en".

08 — Admin Dashboard (localStorage)

The admin dashboard at /admin is a client-side management interface. It reads/writes the following localStorage keys:

Storage KeyTypeDescription
mercbex_products_overrideProduct[]Admin-edited products. Merges with the hardcoded catalog at runtime via getProducts(). Matching IDs override fields; new IDs append.
mercbex_store_settings{ promoBanner: {...} }Store-wide settings including the promo banner configuration (title, description, link, colors, enabled).
mercbex_crop_scansScanRecord[]Diagnosis history from the AI Scanner. Read-only in the admin Scans tab.
order_MBX{id}OrderPer-order records written by the checkout page. The admin Orders tab scans for all order_ prefixed keys.

Product Override Interface

interface ProductOverride {
  id: string;                   // Matches hardcoded ID to override, or new ID to append
  name: string;
  activeIngredient: string;
  formulation: string;
  category: "Insecticide" | "Fungicide" | "Herbicide" | "Rodenticide" | "Plant Growth Regulator";
  price: number;
  originalPrice?: number;
  rating: number;
  reviewCount: number;
  description: string;
  targetPests: string[];
  cropSuitability: string[];
